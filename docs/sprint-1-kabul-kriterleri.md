# Sprint 1 Kabul Kriterleri

## Fonksiyonel

- `GET /health` 200 donmeli.
- `POST /api/auth/register` ve `POST /api/auth/login` endpointleri en az placeholder yanit vermeli.
- `POST /api/listings` ve `GET /api/listings` endpointleri iskelet olarak calismali.
- Web'de `/`, `/musteri`, `/firma`, `/admin` route'lari render edilmeli.

## Teknik

- `pnpm install` ve `pnpm dev` ile servisler acilabilmeli.
- API env validasyonu eksik degiskenlerde acik hata vermeli.
- Prisma schema migration uretebilir durumda olmali.
- Kod tabani TypeScript uyumlu olmali.

## Dokumantasyon
